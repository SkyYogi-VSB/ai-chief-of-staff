# 00 — Architecture

## The Brain + Hands Model

Your CoS has two layers:

- **Brain** — The AI agent. It reads, analyzes, categorizes, drafts, and recommends. This is where judgment lives.
- **Hands** — The tools that touch your systems. Email CLI, calendar API, file system. These are dumb pipes — they fetch and push data but make no decisions.

This separation matters. The Brain is expensive (AI tokens) and smart. The Hands are free and fast. Push as much work as possible into the Hands layer — formatting, filtering, categorization — and reserve the Brain for work that actually requires intelligence: polishing a reply, synthesizing a brief, making a judgment call.

```
┌─────────────────────────────────────────────────┐
│                   YOU (decide)                   │
├─────────────────────────────────────────────────┤
│              BRAIN (AI agent)                    │
│  - Draft replies    - Synthesize briefs          │
│  - Polish writing   - Surface insights           │
├─────────────────────────────────────────────────┤
│              HANDS (tools + scripts)             │
│  - Fetch email      - Read calendar              │
│  - Categorize       - File/archive               │
│  - Format output    - Schedule jobs              │
├─────────────────────────────────────────────────┤
│           YOUR SYSTEMS (email, cal, etc.)        │
└─────────────────────────────────────────────────┘
```

## The Three Rhythms

Your CoS runs on three rhythms. Each one is a habit, not a feature.

### Morning Brief (daily, automated)

Runs before you wake up. When you sit down, a brief is waiting:

- Today's calendar (with prep notes for important meetings)
- Overnight email summary — VIP messages surfaced first
- Open todos carried from yesterday
- Active projects and their next steps

**Time to consume:** 2-3 minutes.
**Time to build:** The brief is generated by a script, not the AI. The AI is only called if you want deeper analysis on something.

### End-of-Day Review (daily, interactive)

A 5-minute conversation with your agent before you close the laptop:

1. Review today's todos — done, roll forward, or discard
2. Quick inbox scan for anything urgent that came in late
3. Preview tomorrow's calendar — flag conflicts and early meetings
4. Set tomorrow's todos

This is where discipline lives. The morning brief shows you the world; the EOD review shapes tomorrow.

### Weekly Review (weekly, interactive)

A 20-minute session, ideally Friday:

1. Walk each workstream — what moved, what's stuck, what needs intervention
2. Close completed actions, add new ones from the week
3. Update workstream health
4. Set top 3 priorities for next week

The weekly review is the only time you zoom out. Everything else is execution.

## Platform-Agnostic by Design

This system works with any combination of:

| Layer | Options |
|-------|---------|
| **Email** | Outlook, Gmail, Fastmail, any IMAP client |
| **Calendar** | Outlook, Google Calendar, Apple Calendar |
| **AI Agent** | Claude, ChatGPT, Gemini, Cursor, Windsurf, any CLI agent |
| **Automation** | cron (Linux/Mac), Task Scheduler (Windows), launchd (Mac) |
| **Storage** | Local markdown files (recommended), Notion, Obsidian, plain text |

The scripts in this repo use shell + Python because they're universal. But the *methodology* doesn't care about your stack. If you prefer Node.js or Go, the patterns translate directly.

## Credit Optimization

AI tokens cost money. A well-designed CoS minimizes AI spend by pushing work to free local scripts:

| Operation | Engine | Cost |
|-----------|--------|------|
| Calendar formatting | Local script | Free |
| Email categorization | Local script (keyword + rules) | Free |
| Bulk email actions | Email CLI/API | Free |
| Brief/triage composition | Local script (templates) | Free |
| **Reply polishing** | **AI agent** | **Only AI cost** |
| **Synthesis & insight** | **AI agent** | **Only AI cost** |

In practice, my CoS uses AI tokens only when I'm actively drafting replies or asking for analysis. The daily brief, triage, and weekly review are all generated locally.

## The Safety Contract

Three rules that never bend:

1. **Drafts only.** The CoS never sends email, accepts calendar invites, or posts messages. It creates drafts for your review.
2. **No secrets in prompts.** API keys, passwords, and tokens live in environment variables or keychains, never in prompt text or config files.
3. **Audit trail.** Every action the CoS takes is logged. You can always see what it did and why.
